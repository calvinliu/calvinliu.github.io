简单安全的Linux设置1

发表于 2015-08-20   |   分类于 计算机   |  

写在前面

之前开发的项目多数部署在内网中，即使部署在公网上，也有硬件防火墙作为屏障，系统安全性不需要过多的去担忧。
这次买了VPS，自己作为使用者和管理者不得不考虑安全的问题，下面的系统安全配置对于专业的运维工程师来说是小儿科，甚至还有漏洞，不过对我目前来说基本够用了，遇到好的方案后面再补充啦！

配置

1.使用 iptables 开启常规端口

一般我们只需要开启常规服务端口,命令语法如下:
iptables -A INPUT -p tcp --dport 22 -j ACCEPT
-A: 追加一个新的规则
-p tcp: tcp协议的扩展
—dport 22: 指定一个端口
-j ACCEPT: 接受

清空 iptables 默认规则:
iptables -F

允许 22 端口（作为SSH访问使用）进入和返回:

iptables -A INPUT -p tcp --dport 22 -j ACCEPT
iptables -A OUTPUT -p tcp --sport 22 -m state --state ESTABLISHED -j ACCEPT

允许本机访问（测试使用）:

iptables -A INPUT -s 127.0.0.1 -d 127.0.0.1 -j ACCEPT
iptables -A OUTPUT -s 127.0.0.1 -d 127.0.0.1 -j ACCEPT

允许所有 IP 访问 80 和 443 端口（作为Http和Https使用）进入和返回:

iptables -A INPUT -p tcp -s 0/0 --dport 80 -j ACCEPT
iptables -A OUTPUT -p tcp --sport 80 -m state --state ESTABLISHED -j ACCEPT
iptables -A INPUT -p tcp -s 0/0 --dport 443 -j ACCEPT
iptables -A OUTPUT -p tcp --sport 443 -m state --state ESTABLISHED -j ACCEPT

允许 53 端口（作为DNS服务使用）进入和返回:

iptables -A OUTPUT -p udp --dport 53 -j ACCEPT
iptables -A INPUT -p udp --sport 53 -j ACCEPT

保存配置:
iptables-save

重新加载 iptables:
iptables -L

2.使用 fail2ban 屏蔽扫描SSH端口的IP

安装（ubuntu）:
apt-get install fail2ban -y
查看日志
cat /var/log/fail2ban.log
